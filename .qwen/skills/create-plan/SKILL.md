# Create Plan Skill

Generate structured action plans for tasks and emails.

## When to Use

- New item appears in `/Needs_Action/`
- Need to break down complex tasks
- Determine approval requirements
- Create audit trail for decision-making

## CLI Usage

```bash
# Process all files in Needs_Action
python ai-employee/src/skills/create_plan.py \
  --process \
  --vault "D:\Hackathon-0\AI_Employee_Vault"

# Process single file
python ai-employee/src/skills/create_plan.py \
  --file "D:\Hackathon-0\AI_Employee_Vault\Needs_Action\EMAIL_123.md" \
  --vault "D:\Hackathon-0\AI_Employee_Vault"

# Dry run (preview plan without creating)
python ai-employee/src/skills/create_plan.py \
  --file "D:\Hackathon-0\AI_Employee_Vault\Needs_Action\EMAIL_123.md" \
  --vault "D:\Hackathon-0\AI_Employee_Vault" \
  --dry-run
```

## Plan Templates

### Email Plan

```markdown
---
type: plan
source_type: email
from: client @example.com
subject: Invoice Request
created: 2026-03-29T10:30:00Z
status: pending
requires_approval: true
---

# Action Plan: Process Email

## Objective
Process and respond to email from client @example.com

## Steps

1. [ ] Read and understand email content
2. [ ] Determine required response or action
3. [ ] Draft response or take action
4. [ ] Create approval request (if sending email)
5. [ ] Wait for human approval
6. [ ] Send response via email MCP
7. [ ] Log transaction
8. [ ] Archive email after completion

## Timeline
- **Created:** 2026-03-29 10:30
- **Due:** 2026-03-29 18:00 (same day)
- **Priority:** High

## Approval Required
**Action:** send_email
**Details:** Response to client regarding invoice

---
*Generated by AI Employee Plan Generator*
```

### WhatsApp Message Plan

```markdown

type: plan

## Steps
```

### Payment Plan

```markdown
---
type: plan
source_type: payment
amount: 500.00
recipient: Vendor ABC
created: 2026-03-29T10:30:00Z
status: pending
requires_approval: true
---

# Action Plan: Process Payment

## Objective
Process payment of $500.00 to Vendor ABC

## Steps

1. [ ] Verify payment details and amount
2. [ ] Check available funds
3. [ ] Create payment approval request
4. [ ] Wait for human approval
5. [ ] Execute payment via MCP
6. [ ] Log transaction and update records

## Approval Required
**Action:** payment_execute
**Amount:** $500.00
**Recipient:** Vendor ABC
```

## Plan Generation Logic

### Priority Classification

| Source | High Priority | Medium | Low |
|--------|--------------|--------|-----|
| **Email** | invoice, payment, urgent | meeting, schedule | newsletter |
| **WhatsApp** | urgent, asap, help | question, update | general chat |
| **Payment** | > $1000 | $100-1000 | < $100 |

### Approval Requirements

| Action Type | Always Requires Approval |
|-------------|-------------------------|
| `email_send` | Yes - All external emails |
| `payment` | Yes - All payments |
| `linkedin_post` | Yes - All posts |
| `file_delete` | Yes - Permanent deletions |
| `archive_only` | No - Auto-approve |

## Integration

### Called By

- **Orchestrator**: Process new items
- **Gmail Watcher**: After email detected
- **WhatsApp Watcher**: After message detected

### Calls

- [`create_approval_request`](../create-approval-request/SKILL.md) - For sensitive actions
- [`log_action`](../log-action/SKILL.md) - Log plan creation
- **File System**: Move file to `/Plans/`

## Error Handling

| Error | Behavior | Recovery |
|-------|----------|----------|
| Invalid source file | Logs error, skips | Fix source file format |
| No template match | Uses generic template | Add custom template |
| Cannot create file | Logs error | Check permissions |
| Move fails | Logs error, keeps in place | Check if file open |

## Troubleshooting

### Plan not being created

**Check:** Source file has valid frontmatter

**Fix:**
```markdown
---
type: email
from: sender @example.com
---
```

### Wrong template used

**Check:** Source type in frontmatter

**Fix:** Ensure correct type:
```markdown
---
type: whatsapp_message
---
```
