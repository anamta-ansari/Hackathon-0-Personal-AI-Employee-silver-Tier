## 📌 Project Overview

The AI Employee is an autonomous agent system that operates 24/7 to manage personal and business affairs through a **Perception → Reasoning → Action** loop:

| Phase | Component | Description |
|-------|-----------|-------------|
| **Perceive** | Watchers | Monitor Gmail, WhatsApp, file systems, and other inputs via lightweight Python scripts |
| **Reason** | Qwen Code | Analyze inputs, create action plans, and make decisions |
| **Act** | MCP Servers | Execute actions (send emails, post to LinkedIn, make payments) through Model Context Protocol |
| **Remember** | Obsidian Vault | Store everything locally for transparency, audit, and long-term memory |

### What It Can Do

| Category | Capabilities |
|----------|-------------|
| **Email** | Monitor Gmail, process emails, send responses (with human approval) |
| **Social Media** | Auto-generate and post to LinkedIn with business content |
| **Messaging** | Monitor WhatsApp for urgent keywords ("urgent", "invoice", "payment") |
| **Workflow** | Human-in-the-loop approval system for all sensitive actions |
| **Scheduling** | Run tasks on schedule (daily briefings, weekly audits) |
| **Audit** | Complete logging of all actions for compliance and review |

### Key Concept: Digital FTE

A **Digital FTE** (Full-Time Equivalent) is an AI agent built and priced like a human employee:

| Metric | Human FTE | Digital FTE |
|--------|-----------|-------------|
| Availability | 40 hrs/week | 168 hrs/week (24/7) |
| Cost per Task | ~$3.00–$6.00 | ~$0.25–$0.50 |
| Annual Hours | ~2,000 | ~8,760 |
| Consistency | 85–95% | 99%+ |

## 🛠 Tech Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| **AI Reasoning** | [Qwen Code](https://agentfactory.panaversity.org/) | Primary reasoning engine for analysis, planning, and decision-making |
| **Knowledge Base** | [Obsidian](https://obsidian.md/) | Local-first Markdown storage for dashboard, handbook, goals, and audit logs |

| **Version Control** | Git + GitHub Desktop | Vault sync (Gold/Platinum tier) and code management |
| **Process Management** | PM2 / Custom Watchdog | Keep watchers alive, auto-restart on failure |

## 🧑‍💻 Coding Conventions

### Python Style Guide

| Rule | Convention |
|------|------------|
| **Formatting** | PEP 8 compliant, 4-space indentation |
| **Line Length** | Max 120 characters |
| **Imports** | Standard library → Third-party → Local (grouped, sorted) |
| **Type Hints** | Required for all function signatures |
| **Docstrings** | Google-style for all public functions/classes |
| **Naming** | `snake_case` for functions/variables, `PascalCase` for classes |

### File & Module Organization

```python
# Standard library
import os
import time
from pathlib import Path
from abc import ABC, abstractmethod

# Third-party
from playwright.sync_api import sync_playwright

# Local
from base_watcher import BaseWatcher
from utils.logger import get_logger
```

### Error Handling

| Pattern | Usage |
|---------|-------|
| **Specific Exceptions** | Catch `TransientError`, `AuthError`, etc. — never bare `except:` |
| **Retry Logic** | Use exponential backoff for transient failures (max 3 attempts) |
| **Graceful Degradation** | Queue actions locally when external APIs are unavailable |
| **Logging** | Log all errors with context to `/Vault/Logs/` |

### Security Rules

| Rule | Enforcement |
|------|-------------|
| **No hardcoded secrets** | Use `.env` or OS secrets manager |
| **Never commit credentials** | `.env`, `credentials.json`, `token.json` in `.gitignore` |
| **Dry-run support** | All action scripts support `--dry-run` flag |
| **Human-in-the-loop** | Sensitive actions (payments, emails to new contacts) require approval files |
| **Audit logging** | Every action logged with timestamp, actor, target, and result |

### Agent Skills Structure

```
    └── <skill>.py    # Main skill logic
```


Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`

## ▶️ Key Commands

### Setup & Installation

```bash
# Install Python dependencies
cd ai-employee
pip install -r requirements.txt

# Install Playwright browsers
playwright install chromium

# Set up environment
copy .env.example .env   # Windows
# Edit .env with your credentials
```

### Running the System

```bash
cd D:\Hackathon-0\ai-employee

# Start orchestrator (main automation loop)
python src/orchestration/orchestrator.py

# Dry-run mode (safe testing - no side effects)
python src/orchestration/orchestrator.py --dry-run

# Start individual watchers
python src/watchers/gmail_watcher.py
python src/watchers/filesystem_watcher.py
```

### Email Automation

```bash
# Authenticate Gmail (first-time)
python src/skills/gmail-auth.py auth

# Check authentication status
python src/skills/gmail-auth.py status

# Process emails in Needs_Action
python src/skills/process-email.py

# Send email (with approval workflow)
python src/skills/send-email.py --to "client@example.com" --subject "Invoice #123" --body "Please find attached."

# Dry-run email send (preview only)
python src/skills/send-email.py --dry-run --to "client@example.com" --subject "Test" --body "Test"
```

### LinkedIn Automation

```bash
# Authenticate LinkedIn (first-time)
python src/skills/linkedin-auth.py login

# Check session status
python src/skills/linkedin-auth.py status

# Generate post content from Business_Goals.md
python src/skills/linkedin-content-generator.py

# Preview generated content
python src/skills/linkedin-content-generator.py --preview

# Create LinkedIn post (generates approval request)
python src/skills/linkedin-post.py --content "Your post content here"

# Browser-based posting (Playwright)
python src/skills/linkedin-browser-post.py --content "Your post content here"

# Test LinkedIn connection
python src/skills/linkedin-browser-post.py --test
```

### WhatsApp Monitoring

```bash
# Start WhatsApp watcher (monitors for keywords)
python src/watchers/whatsapp_watcher.py

# Session management handled automatically via Playwright
```

### Approval Workflow

```bash
# Create approval request
python src/skills/create-approval-request.py --action "send_email" --target "client@example.com"

# Review pending approvals
dir AI_Employee_Vault\Pending_Approval\   # Windows

# Approve: move file to Approved/
move AI_Employee_Vault\Pending_Approval\file.md AI_Employee_Vault\Approved\

# Reject: move file to Rejected/
move AI_Employee_Vault\Pending_Approval\file.md AI_Employee_Vault\Rejected\
```

### Dashboard & Reporting

```bash
# Update dashboard
python src/skills/update-dashboard.py

# Generate weekly CEO briefing
python src/skills/update-dashboard.py --briefing

# View audit logs
type AI_Employee_Vault\Logs\actions.json   # Windows
```

### Scheduling (Windows)

```batch
# Schedule daily briefing at 8:00 AM
schtasks /Create /TN "AI_Employee_Briefing" /TR "python D:\Hackathon-0\ai-employee\src\skills\update-dashboard.py --briefing" /SC DAILY /ST 08:00

# Schedule LinkedIn posting every Monday at 9:00 AM
schtasks /Create /TN "AI_Employee_LinkedIn" /TR "python D:\Hackathon-0\ai-employee\src\skills\linkedin-content-generator.py --auto" /SC WEEKLY /D MON /ST 09:00

# View scheduled tasks
schtasks /Query /FO LIST
```

### Process Management

```bash
# Using PM2 (recommended for 24/7 operation)
npm install -g pm2

# Start orchestrator with PM2
pm2 start src/orchestration/orchestrator.py --interpreter python --name "ai-orch"

# Start watchers with PM2
pm2 start src/watchers/gmail_watcher.py --interpreter python --name "gmail-watch"

# Save process list (auto-start on reboot)
pm2 save
pm2 startup

# Monitor processes
pm2 status
pm2 logs ai-orch
```

### Testing & Debugging

```bash

cd ai-employee

python src/orchestration/orchestrator.py --log-level DEBUG
```
